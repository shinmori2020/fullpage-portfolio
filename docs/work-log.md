### レスポンシブ基盤整備

1. `sections-responsive.css` 作成 + `index.html` に読み込み追加
2. ユーティリティクラス定義（sp-hide, sp-bg-image, sp-overlay, sp-text-white等）
3. ナビドット・プログレスバー: 768px以下で非表示
4. セクションカウンター: SP時は中央配置
5. タッチ感度: 768px以下で50px→80pxに引き上げ
6. ベースフォントサイズ rem化（PC 16px → 1024px 15px → 768px 14px → 480px 12px）
7. プレースホルダー: SP時にサイズ縮小（position: static化）

### 試して不採用にした方式

| 方式 | 不採用理由 |
|---|---|
| zoom方式 | 文字が小さくなりすぎ、縦長レイアウトが解決しない |
| scale方式（transform: scale + JS） | 縮小後に白い余白が残る |
| 全スライド1カラム化 | 100vh固定のためコンテンツが収まらない |

### レスポンシブで学んだ教訓

1. `position: absolute` → `position: static !important` で確実にリセット
2. 共通クラス（.placeholder-image）の変更は全スライドに影響する → 個別対応が安全
3. 装飾帯をlayout外に配置すると `order` が効かない → HTML構造の修正が必要
4. `flex: 1` が縦並び時にカードの高さを広げる → `flex: none` で解決
5. 詳細度の競合は `.panel.slide-N >` セレクタで解決
6. スライドごとに方式を選択すべき（全スライド統一は困難）
